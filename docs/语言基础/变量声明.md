# 变量声明

JavaScript 中有三种变量声明方式：`var`、`let` 和 `const`，其中 `var` 和另外两个声明方式存在显著差异。掌握它们的区别对于编写健壮、可维护的代码至关重要。

## 详细说明

### var 声明

`var` 是早期 JavaScript 的变量声明方式，主要特点是

- 使用 [**函数作用域**](../其他知识点/作用域类型.md#函数作用域)
- 变量会被 [**提升**](../其他知识点/提升.md#变量提升)，并在声明前可读到 `undefined`
- 可以重复声明同名变量

```js
console.log(age); // undefined，变量被提升但未初始化
var age = 18;
console.log(age);
```

### let 声明

`let` 用于声明“后续可能会变化”的变量

- 使用 [**块级作用域**](../其他知识点/作用域类型.md#块级作用域)
- 存在 [**暂时性死区**](../其他知识点/提升.md#暂时性死区)，声明前访问会报错
- 不允许在同一作用域重复声明

```js
// [!code error] 在 let 声明之前访问变量会抛出 ReferenceError
console.log(n); // ReferenceError: Cannot access 'n' before initialization
let n = 0;
n += 1;
console.log(n); // 1
```

### const 声明

`const` 用于声明“绑定不变”的变量（常量，constant）

- 使用 [**块级作用域**](../其他知识点/作用域类型.md#块级作用域)
- 存在 [**暂时性死区**](../其他知识点/提升.md#暂时性死区)，声明前访问会报错
- 不允许在同一作用域重复声明
- 声明时必须初始化
- 不能重新赋值，但对象/数组的内容仍可修改

```js
// 在 const 声明之前访问变量会抛出 ReferenceError
console.log(p); // [!code error] ReferenceError: Cannot access 'p' before initialization
const p = { name: "Alice" };
// 可以修改对象内容
p.name = "Bob";
// 重新赋值会抛出 TypeError
p = {}; // [!code error] TypeError: Assignment to constant variable.
// 声明时必须初始化，否则会抛出 SyntaxError
const name; // [!code error] SyntaxError: Missing initializer in const declaration
```

### 表格总览

| 特性       | var                      | let            | const                 |
| ---------- | ------------------------ | -------------- | --------------------- |
| 作用域     | 函数作用域               | 块级作用域     | 块级作用域            |
| 变量提升   | 提升并初始化为 undefined | 处于暂时性死区 | 处于暂时性死区        |
| 重复声明   | ✅ 允许                  | ❌ 禁止        | ❌ 禁止               |
| 重新赋值   | ✅ 允许                  | ✅ 允许        | ❌ 禁止（引用不可变） |
| 必须初始化 | ❌ 否                    | ❌ 否          | ✅ 是                 |

## 使用建议

现代 JavaScript 开发中建议统一使用 `let` 和 `const`，避免 `var`，主要原因如下：

1. `var` 是 [函数作用域](../其他知识点/作用域类型.md#函数作用域)，容易出现变量“泄漏”
2. `var` 可重复声明，容易被无意覆盖
3. `var` 提升为 `undefined`，更容易产生隐蔽错误

### 变量提升和作用域混乱问题

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 输出 3,3,3
}
```

```js
let theme = "light"; // 全局配置

function applyTheme() {
  // 由于 var 提升，函数顶部隐式存在 var theme;
  if (true) {
    var theme = "dark"; // 实际修改的是函数作用域内的 theme
    console.log("临时主题:", theme); // "dark"
  }
  // 此处访问的 theme 是函数作用域内的变量，全局 theme 被遮蔽了
  console.log("最终应用的主题:", theme); // "dark" —— 全局配置被意外覆盖！
}

applyTheme();
```

### 变量提升和覆盖问题

在 `for` 循环或 `if` 语句等块级结构中使用 `var` 声明的变量会被提升到函数作用域的顶部，导致变量在预期之外的地方可访问。假如循环内部出现了和循环外部同名的变量，可能会无意中覆盖外部变量，造成难以发现的 bug。

```js
var starItem = [{}, {}, {}, {}, {}]; // 假设此处是 5 个影片剪辑
var isStarActive = [0, 0, 0, 0, 0]; // 0 表示未激活，1 表示激活

// 函数要求根据 index 更新 isStarActive，并激活对应的 starItem
function lightStar(index) {
  // 假设传入的是 1-5，所以在这里统一处理为 0-4
  // 但是使用了 i 作为变量名
  var i = index - 1; // [!code warning]
  isStarActive[i] = 1; // 激活对应的星星

  // [!code warning] 在这里需要遍历数组，但错误地使用了 i 作为循环变量
  for (var i = 0; i < 5; i++) {
    starItem[i].gotoAndStop(0);
  }

  // 抛出错误，因为 i 已经被循环覆盖了，此时 i 的值是 5，超出数组范围
  // starItem[5] 是 undefined，调用 gotoAndStop 会抛出 TypeError
  starItem[i].gotoAndStop(1); // [!code error] TypeError: Cannot read properties of undefined
}
```

## 实践推荐

- 默认使用 `const`
- 变量需要重新赋值时使用 `let`
- 避免使用 `var`
