# JavaScript 中的变量声明

## 学习目标

- 理解 `let`、`const`、`var` 的语义差异
- 能根据场景正确选择变量声明方式
- 理解为什么现代开发中不推荐使用 `var`

## 三种声明方式速览

| 关键字  | 作用域     | 是否可重复声明 | 是否可重新赋值       | 是否有暂时性死区（TDZ） | 提升行为                       |
| ------- | ---------- | -------------- | -------------------- | ----------------------- | ------------------------------ |
| `var`   | 函数作用域 | 可以           | 可以                 | 否                      | 声明提升，初始值为 `undefined` |
| `let`   | 块级作用域 | 不可以         | 可以                 | 是                      | 声明提升，但未初始化           |
| `const` | 块级作用域 | 不可以         | 不可以（引用不可变） | 是                      | 声明提升，但未初始化           |

## 详细说明

### 1. `var`

`var` 是早期 JavaScript 的变量声明方式，主要特点是：

- 使用函数作用域，不受 `{}` 代码块限制
- 可以重复声明同名变量
- 变量会被提升，并在声明前可读到 `undefined`

```js
if (true) {
  var age = 18;
}

console.log(age); // 18（泄漏到块外）
```

### 2. `let`

`let` 用于声明“后续可能会变化”的变量：

- 块级作用域，仅在当前 `{}` 内有效
- 不允许在同一作用域重复声明
- 存在暂时性死区，声明前访问会报错

```js
let count = 0;
count += 1;

console.log(count); // 1
```

### 3. `const`

`const` 用于声明“绑定不变”的变量：

- 块级作用域
- 声明时必须初始化
- 不能重新赋值，但对象/数组的内容仍可修改

```js
const user = { name: "Alice" };
user.name = "Bob"; // 合法：修改对象内容

// user = {} // 非法：不能重新绑定
```

## 为什么不使用 `var`

现代 JavaScript 开发中建议统一使用 `let` 和 `const`，避免 `var`，主要原因如下：

1. `var` 是函数作用域，容易出现变量“泄漏”

```js
if (true) {
  var msg = "hello";
}
console.log(msg); // hello
```

很多开发者直觉上会认为 `if` 块内变量应该在块外不可见，`var` 会打破这种直觉，增加维护难度。

2. `var` 可重复声明，容易被无意覆盖

```js
var total = 100;
var total = 200; // 不报错，可能导致逻辑被悄悄改写
```

3. `var` 提升为 `undefined`，更容易产生隐蔽错误

```js
console.log(price); // undefined，不会立刻报错
var price = 99;
```

这种“先用后声明”不会马上失败，反而会把问题延后到业务逻辑中，排查成本更高。

4. 团队协作中一致性更差

当项目同时混用 `var`、`let`、`const` 时，代码规范变得不统一，评审和重构成本都会上升。

## 推荐实践

- 默认使用 `const`
- 变量需要重新赋值时使用 `let`
- 避免使用 `var`
- 开启 ESLint 规则（例如 `no-var`、`prefer-const`）约束代码风格

## 小结

`let` 和 `const` 提供了更清晰、更安全的作用域与初始化语义，能显著降低由提升和变量覆盖引发的问题。除非维护历史遗留代码，否则不建议在新代码中使用 `var`。
