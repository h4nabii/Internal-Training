# 变量类型

JavaScript 是一门动态类型语言，变量可以随时赋予不同类型的值。TypeScript 在 JavaScript 之上引入了静态类型检查，允许我们显式声明变量、参数、返回值的预期类型，从而在编码阶段发现潜在错误。

## 基础类型

JavaScript 的 7 种基础类型（Primitive Types）如下：

- `string`：字符串类型，用于表示文本数据
- `number`：数字类型，用于表示整数和浮点数
- `boolean`：布尔类型，只有两个值 `true` 和 `false`
- `null`：表示“无”或“空”的值，只有一个值 `null`
- `undefined`：表示变量未定义或未赋值，只有一个值 `undefined`
- `symbol`：ES6 加入，符号类型，表示独一无二的值，常用于对象属性的唯一标识
- `bigint`：ES6 加入，大整数类型，可用于表示超过 `Number.MAX_SAFE_INTEGER` 的整数

基础类型的值本身是不可变的。任何看似“修改”的操作，实际上是返回了一个全新的值。

```js
let str = "Hello";
str[0] = "h"; // 静默失败
console.log(str); // "Hello"
```

TypeScript 中可以显式声明变量的类型，对于简单的基础类型，TypeScript 还支持类型推断，可以根据初始值自动推断出变量的类型：

```ts
let message = "Hello, TypeScript"; // TypeScript 会推断 message 的类型为 string
let count = 42; // TypeScript 会推断 count 的类型为 number

// 显式声明函数参数和返回值的类型
function greet(name: string): string {
  return `Hello, ${name}!`;
}

// 函数返回值的类型通常可以省略，TypeScript 会根据函数体自动推断
function add(a: number, b: number) {
  return a + b; // TypeScript 会推断返回值的类型为 number
}
```

::: info

TypeScript 中除了基本的 7 种基础类型和 3 种引用类型，还有一些特殊的类型，如 `any`（任意类型）、`unknown`（未知类型）、`never`（永远不会发生的类型）等，这些感兴趣的可以自行查阅文档。

在此只提一点，不要滥用 `any` 类型，因为它会完全放弃 TypeScript 的类型检查，让 TypeScript 变成 AnyScript，失去类型系统的意义。

:::

使用 typeof 可以获取运行时类型：

```js
console.log(typeof "Hello"); // "string"
console.log(typeof 42); // "number"
console.log(typeof true); // "boolean"
console.log(typeof null); // "object"（这是 JavaScript 的一个历史遗留问题）
console.log(typeof undefined); // "undefined"
console.log(typeof Symbol("sym")); // "symbol"
console.log(typeof 9007199254740991n); // "bigint"
```

::: warning

需要注意的是，`typeof null` 返回 `"object"` 是 JavaScript 的一个历史遗留问题，虽然 `null` 是一个基础类型，但它被错误地识别为对象类型。这是因为在 JavaScript 的早期实现中，值的类型是通过一个内部标签来标识的，而 `null` 的标签被误设置为对象类型。这个问题已经存在了很长时间，并且由于兼容性原因无法修复。

但在 TypeScript 的类型系统中，null 仍然被视作独立的 null 类型。

:::

## 引用类型

JavaScript 的 3 种引用类型（Reference Types）如下：

- `object`：对象类型，用于表示复杂数据结构，如对象字面量、数组、函数等
- `array`：数组类型，虽然数组也是对象，但它们具有特殊的行为和方法，因此也被单独分类
- `function`：函数类型，虽然函数也是对象，但它们具有可调用的特性，因此被单独分类

引用类型在 JavaScript 中存储的是内存地址引用。TypeScript 也能对其进行基础标注。 在简单场景下，我们可以直接使用 {} 或 [] 来描述结构。

```ts
type User = { name: string; age: number };

// 标注一个普通对象
let user: User = { name: "Alice", age: 25 };

// 标注字符串数组（两种等效写法）
let list1: string[] = ["a", "b", "c"];
let list2: Array<number> = [1, 2, 3];

// 混合类型数组（利用联合类型）
let mixed: (string | number)[] = [1, "two", 3];
```

引用类型的内容是可变的。即使使用 const 声明，对象内部的属性或数组内部的元素依然可以被修改。

```ts
const obj = { name: "Alice" };
obj.name = "Bob"; // 修改属性成功
obj = {}; // [!code error] 错误：const 声明的变量不能重新赋值指向新对象

const arr: number[] = [1, 2, 3];
arr[0] = 0; // 修改元素成功
```

TypeScript 的作用：虽然它无法阻止运行时的内容修改（这是 JS 本身的特性），但它可以通过类型标注确保我们给 name 赋的是 string，给 arr 推入的是 number，从而避免类型错乱。

## 联合类型

联合类型（Union Types）是 TypeScript 中最常用、最直观的特性之一。它允许一个变量拥有多种可能的数据类型，使用竖线 `|` 分隔。

```ts
// 变量既可以是字符串，也可以是数字
let id: string | number;
id = "abc123"; // 合法
id = 12345; // 合法
id = true; // [!code error] 类型错误：不能将布尔值赋给 string | number 类型

// 函数参数也支持联合类型
function printId(id: string | number) {
  console.log(`ID: ${id}`);
}
printId("A001");
printId(200);
```

联合类型在实际开发中非常常见，比如 API 返回的状态码可能既支持数字也支持语义化字符串。

### 联合类型与类型收窄

当一个变量被标注为联合类型时，TypeScript 会要求我们先区分具体类型再操作。这称为类型收窄（Type Narrowing）。

```ts
function formatValue(value: string | number) {
  // 直接调用 value.toUpperCase() 会报错，因为 number 没有该方法
  if (typeof value === "string") {
    // 在这个分支里，TypeScript 知道 value 一定是 string
    return value.toUpperCase();
  } else {
    // 这个分支里，value 一定是 number
    return value.toFixed(2);
  }
}
```

这种“先判断类型，再安全使用”的模式确保了代码在运行时不会意外报错。
