# 高阶函数

本章用于建立“会用”到“用对”的高阶函数思维，重点覆盖 `map`、`forEach`、`every` 等数组方法。

## 学习目标

- 理解“高阶函数”的定义与价值
- 能根据场景选择合适的方法（遍历、转换、过滤、归约、查找、判断）
- 掌握常见 API 的返回值、短路行为、副作用风险
- 避免常见坑（空数组、`this`、异步误用、性能误判）

## 1. 概念定义

### 1.1 什么是高阶函数

- 接收函数作为参数，或返回一个函数的函数
- 常用于“把行为作为数据传递”，提升复用性与可组合性

### 1.2 为什么要用

- 声明式表达业务意图，减少样板循环代码
- 更容易组合与复用（小函数拼接）
- 在团队协作中可读性更高

### 1.3 与普通循环的关系

- 不是替代关系，而是“表达方式”不同
- 简单遍历可用循环；数据变换场景优先高阶函数

## 2. 常见方法分组

### 2.1 遍历类：`forEach`

- 典型用途：只执行副作用（打印、埋点、原地写入）
- 返回值：`undefined`
- 短路：不支持（不能 `break`）
- 是否生成新数组：否
- 注意点：不要把 `forEach` 当成可链式转换工具

### 2.2 转换类：`map`

- 典型用途：一对一映射，生成新数组
- 返回值：新数组（长度通常与原数组一致）
- 短路：不支持
- 是否生成新数组：是
- 注意点：回调必须 `return`，否则得到 `undefined` 组成的新数组

### 2.3 过滤类：`filter`

- 典型用途：按条件筛选元素
- 返回值：新数组（长度不固定）
- 短路：不支持（会遍历全部）
- 是否生成新数组：是
- 注意点：回调返回布尔语义值，避免隐式转换过度复杂

### 2.4 归约类：`reduce`

- 典型用途：累加、分组、扁平化、对象构建
- 返回值：任意类型（由累计器决定）
- 短路：不支持
- 是否生成新数组：不一定（取决于累计器）
- 注意点：建议总是提供初始值，避免空数组时报错

### 2.5 查找类：`find` / `findIndex`

- 典型用途：查找首个满足条件的元素或索引
- 返回值：元素 / 索引（未找到时分别为 `undefined` / `-1`）
- 短路：支持（找到即停）
- 是否生成新数组：否
- 注意点：只取第一个命中，若需全部命中应使用 `filter`

### 2.6 判断类：`some` / `every`

- `some`：是否存在至少一个满足条件
- `every`：是否全部满足条件
- 返回值：布尔值
- 短路：均支持（`some` 命中即停，`every` 失败即停）
- 注意点：空数组上 `every` 返回 `true`（真空真值）

## 3. 方法对比速查

| 方法 | 核心语义 | 返回值 | 是否短路 | 是否产生新数组 | 常见误用 |
| --- | --- | --- | --- | --- | --- |
| `forEach` | 执行副作用 | `undefined` | 否 | 否 | 期待 `return` 结果 |
| `map` | 一对一映射 | 新数组 | 否 | 是 | 回调未返回值 |
| `filter` | 条件筛选 | 新数组 | 否 | 是 | 用于“找第一个” |
| `reduce` | 聚合归约 | 任意类型 | 否 | 视实现而定 | 不传初始值 |
| `find` | 找首个元素 | 元素或 `undefined` | 是 | 否 | 误以为返回数组 |
| `some` | 存在性判断 | 布尔值 | 是 | 否 | 与 `includes` 混用 |
| `every` | 全量校验 | 布尔值 | 是 | 否 | 忽略空数组语义 |

## 4. 性能与可读性建议

- 优先可读性：先写清楚意图，再考虑微优化
- 控制链式深度：过长链路可拆成命名中间变量
- 避免无意义中间数组：高频大数据场景可评估 `for...of`
- 有副作用时显式命名回调，降低维护成本
- 性能判断以基准测试为准，不凭直觉下结论

## 5. 常见陷阱清单

- 在 `map` 中忘记 `return`
- 在 `forEach` 中写 `await` 并期待串行控制
- 误解 `every([]) === true`
- `reduce` 对空数组不传初始值导致异常
- 在回调中修改原数组，造成难以追踪的副作用

## 6. 练习题建议

### 6.1 基础题

- 用 `map` 将用户数组转换为用户名数组
- 用 `filter` 筛选出价格大于 100 的商品
- 用 `every` 判断是否所有分数都及格

### 6.2 进阶题

- 用 `reduce` 实现按字段分组（groupBy）
- 用 `some` + `every` 实现权限校验逻辑
- 比较 `find` 与 `filter` 在“首个命中”场景的差异

### 6.3 综合题

- 给定订单列表，完成：筛选有效订单 -> 映射金额 -> 汇总总额
- 约束：至少使用 `filter`、`map`、`reduce`，并说明每步意图
