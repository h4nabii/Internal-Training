# JavaScript 中的函数闭包

## 什么是闭包？

闭包（Closure）指的是：**函数在定义时，能够“记住”并持续访问它所在词法作用域中的变量，即使这个函数在其原始作用域之外执行，依然可以访问这些变量。**

可以先把它理解成一句话：

**函数 + 它创建时所在的作用域环境 = 闭包**

闭包不是某种特殊语法，而是 JavaScript 中函数和词法作用域共同作用后的结果。只要函数能够访问外层作用域中的变量，并且这些变量在函数执行时仍然被保留，这就是闭包。

## 为什么会有闭包？

JavaScript 采用的是 **词法作用域**（Lexical Scope），也就是说：函数能访问哪些变量，不取决于它在哪里调用，而取决于它在哪里定义。

看一个最基本的例子：

```js
function outer() {
  const message = "hello closure";

  function inner() {
    console.log(message);
  }

  return inner;
}

const fn = outer();
fn(); // 输出 hello closure
```

按直觉看，`outer()` 已经执行结束了，它内部的局部变量 `message` 理论上应该被销毁。但 `inner` 仍然能访问到它，这就是闭包的表现。

原因在于：返回出去的 `inner` 还在使用 `message`，因此 JavaScript 引擎不会立即回收相关作用域中的变量。

## 闭包成立的两个关键条件

通常可以这样理解闭包形成的条件：

1. 在一个函数内部创建了另一个函数。
2. 内部函数引用了外部函数作用域中的变量。

例如：

```js
function createCounter() {
  let count = 0;

  return function () {
    count += 1;
    return count;
  };
}

const counter = createCounter();

console.log(counter()); // 1
console.log(counter()); // 2
console.log(counter()); // 3
```

这里返回的匿名函数每次执行时都能访问并修改 `count`，说明 `count` 没有随着 `createCounter()` 执行完毕而消失。

## 闭包到底“保存”了什么？

闭包保存的并不是变量某一时刻的“值的拷贝”，而是对变量的**引用关系**。

这点很重要。

```js
function outer() {
  let count = 0;

  return {
    add() {
      count += 1;
    },
    get() {
      return count;
    },
  };
}

const result = outer();
result.add();
result.add();
console.log(result.get()); // 2
```

`add` 和 `get` 闭包引用的是同一个 `count`，所以一个函数修改后，另一个函数能读到最新结果。

## 闭包与作用域的关系

闭包离不开作用域。理解闭包之前，先要明确几个概念：

1. 全局作用域：最外层可访问的作用域。
2. 函数作用域：函数内部形成的作用域。
3. 块级作用域：由 `let`、`const` 配合代码块形成的作用域。
4. 词法作用域：函数在定义时就确定了可访问的外部变量范围。

闭包本质上就是词法作用域的延续。函数把定义时可访问的那部分作用域“带”了出去。

```js
const name = "global";

function outer() {
  const name = "outer";

  return function inner() {
    console.log(name);
  };
}

const fn = outer();
fn(); // 输出 outer，而不是 global
```

这里 `inner` 访问的是定义位置所在作用域中的 `name`，而不是调用位置附近的同名变量。

## 闭包的典型应用场景

### 1. 封装私有变量

闭包最常见的用途之一，是模拟“私有变量”。

```js
function createBankAccount(initialBalance) {
  let balance = initialBalance;

  return {
    deposit(amount) {
      balance += amount;
    },
    withdraw(amount) {
      if (amount > balance) {
        console.log("余额不足");
        return;
      }
      balance -= amount;
    },
    getBalance() {
      return balance;
    },
  };
}

const account = createBankAccount(100);
account.deposit(50);
account.withdraw(30);
console.log(account.getBalance()); // 120
console.log(account.balance); // undefined
```

`balance` 无法从外部直接访问，只能通过暴露出来的方法进行操作。

### 2. 函数工厂

闭包很适合根据不同配置生成不同函数。

```js
function createMultiplier(multiplier) {
  return function (value) {
    return value * multiplier;
  };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(10)); // 20
console.log(triple(10)); // 30
```

这里 `double` 和 `triple` 都是闭包，但它们记住的是各自不同的 `multiplier`。

### 3. 回调函数中保留上下文数据

很多异步逻辑、本地缓存、事件处理，本质上都在使用闭包。

```js
function createLogger(prefix) {
  return function (message) {
    console.log(`[${prefix}] ${message}`);
  };
}

const info = createLogger("INFO");
const error = createLogger("ERROR");

info("启动成功");
error("请求失败");
```

### 4. 防抖与节流

前端中常见的防抖、节流函数，通常也依赖闭包保存定时器状态。

```js
function debounce(fn, delay) {
  let timer = null;

  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}
```

这里返回的新函数始终可以访问 `timer`，因此多次调用之间可以共享同一个定时器变量。

## 一个经典面试题

下面这段代码会输出什么？

```js
for (var i = 0; i < 3; i++) {
  setTimeout(function () {
    console.log(i);
  }, 0);
}
```

结果是：

```js
3;
3;
3;
```

原因是：

1. `var` 没有块级作用域，循环中的 `i` 是同一个变量。
2. `setTimeout` 中的回调形成闭包，引用的是同一个 `i`。
3. 当定时器真正执行时，循环已经结束，此时 `i === 3`。

### 用 let 修复

```js
for (let i = 0; i < 3; i++) {
  setTimeout(function () {
    console.log(i);
  }, 0);
}
```

输出：

```js
0;
1;
2;
```

因为 `let` 会为每次循环创建一个新的块级作用域，每个回调闭包拿到的都是各自独立的 `i`。

### 用 IIFE 修复

如果必须使用 `var`，也可以借助 IIFE 人为创建作用域：

```js
for (var i = 0; i < 3; i++) {
  (function (current) {
    setTimeout(function () {
      console.log(current);
    }, 0);
  })(i);
}
```

这里 IIFE 的参数 `current` 会形成独立闭包，因此结果同样是 `0 1 2`。

## 闭包的优点

闭包很强大，常见优点有：

1. 可以延长变量的生命周期。
2. 可以创建私有状态，减少全局变量污染。
3. 可以实现函数工厂、柯里化、偏函数等高级用法。
4. 在异步回调、事件处理、缓存设计中非常实用。

## 闭包的风险与注意事项

闭包并不神秘，但不能滥用。

### 1. 可能造成不必要的内存占用

如果闭包长期引用某些本该释放的大对象，那么这些对象就无法被垃圾回收。

```js
function createHandler() {
  const bigData = new Array(1000000).fill("data");

  return function () {
    console.log(bigData.length);
  };
}
```

只要返回的函数还存在，`bigData` 就可能一直留在内存里。

::: warning

闭包本身不是内存泄漏，但**不合理地持有不再需要的数据**，就可能导致内存长期无法释放。

:::

### 2. 容易因为共享变量产生意外结果

如果多个闭包共享同一个外部变量，而你没有意识到这一点，就容易写出难排查的问题。

```js
function createActions() {
  let value = 0;

  return {
    inc() {
      value += 1;
    },
    dec() {
      value -= 1;
    },
    get() {
      return value;
    },
  };
}
```

这段代码本身没问题，但你必须明确：三个方法操作的是同一个 `value`。

## 如何判断自己是不是写了闭包？

可以快速自检这两个问题：

1. 一个函数是否访问了它外层作用域中的变量？
2. 这个函数是否在外层函数执行结束后，依然可能被调用？

如果答案都是“是”，那么大概率就是闭包。

## 闭包和普通嵌套函数的区别

并不是所有嵌套函数都一定需要重点讨论为“闭包问题”。

```js
function outer() {
  const num = 1;

  function inner() {
    console.log(num);
  }

  inner();
}
```

这段代码中，`inner` 也能访问 `num`，从语言机制上说依然涉及闭包。但在日常开发里，大家更强调的是：**当内部函数离开原始作用域后，仍然访问外部变量** 的这种情况，因为这时闭包的效果更明显。

## 总结

闭包的核心并不复杂，可以概括为三点：

1. 闭包来自函数和词法作用域。
2. 内部函数可以持续访问外部函数中的变量。
3. 闭包常用于封装状态，但也要注意内存和共享变量问题。

如果你把闭包只记成一句术语，往往很快就忘了；但如果你记住它最重要的用途是：**让函数“带着数据一起走”**，那理解会稳定很多。

::: info

学习闭包时，不要只背定义。建议你重点练习下面三类代码：

1. 计数器。
2. 循环 + 异步回调。
3. 防抖 / 节流。

把这三类例子真正写明白，闭包基本就算掌握了。

:::
