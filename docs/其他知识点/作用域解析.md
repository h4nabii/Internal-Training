# 作用域解析

## 作用域链

当访问一个变量时，JS 会先在当前作用域查找；找不到就向外层作用域查找，直到全局作用域。这条查找路径就是作用域链。

```js
const name = "global";

function outer() {
  const name = "outer";

  function inner() {
    console.log(name); // outer
  }

  inner();
}

outer();
```

`inner` 先找自己的作用域，没有，再找 `outer`，最后才会找全局。

## 词法作用域（静态作用域）

JavaScript 使用词法作用域：变量能否访问，由“函数定义的位置”决定，而不是“函数调用的位置”决定。

```js
const value = "global";

function show() {
  console.log(value);
}

function run() {
  const value = "local";
  show(); // 仍然是 global
}

run();
```

`show` 定义在全局位置，所以它读取的是全局的 `value`。

## 作用域与闭包的关系

闭包本质上是“函数 + 定义时的词法环境”。函数执行结束后，内部函数仍然可以访问外层作用域中的变量。

```js
function createCounter() {
  let count = 0;
  return function () {
    count += 1;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

这里返回的函数形成了闭包，持续引用 `count`。
