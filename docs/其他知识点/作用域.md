# 作用域

## 学习目标

- 理解什么是作用域，以及它为什么影响变量可访问性
- 区分全局作用域、函数作用域、块级作用域
- 理解暂时性死区（TDZ）和 `let`/`const` 的行为
- 知道作用域链、词法作用域与闭包之间的关系

## 什么是作用域

作用域（Scope）是变量和函数的可访问范围。你可以把它理解为“名字在代码中生效的区域”。

- 在作用域内：可以访问该标识符
- 在作用域外：无法访问，通常会报错或得到 `undefined`（取决于声明方式与位置）

## 全局作用域

在最外层声明的变量处于全局作用域，几乎在任何地方都能访问。

```js
const siteName = "training";

function printSite() {
  console.log(siteName); // training
}
```

注意：

- 过多全局变量会造成命名冲突和维护困难
- 在浏览器里，`var` 声明的全局变量会挂到 `window` 上，`let`/`const` 不会

## 函数作用域

使用 `var` 声明的变量只在函数内部有效（没有块级作用域特性）。

```js
function demo() {
  var message = "hello";
  console.log(message); // hello
}

// console.log(message); // ReferenceError
```

`var` 的可见范围是“整个函数”，而不是最近的一对 `{}`。

## 块级作用域

使用 `let` 和 `const` 声明的变量具有块级作用域，即只在最近的 `{}` 内可用。

```js
if (true) {
  let a = 1;
  const b = 2;
  console.log(a, b); // 1 2
}

// console.log(a); // ReferenceError
// console.log(b); // ReferenceError
```

常见块：`if`、`for`、`while`、单独的 `{}`。

## 暂时性死区

> 暂时性死区常写作 TDZ，即 Temporal Dead Zone

从块级作用域开始，到变量声明语句执行之前，这段区域叫暂时性死区。

```js
{
  // console.log(x); // ReferenceError（处于 TDZ）
  let x = 10;
  console.log(x); // 10
}
```

要点：

- `let`/`const` 在声明前不可访问
- 这是为了减少“先使用后声明”导致的隐蔽错误

## 作用域链

当访问一个变量时，JS 会先在当前作用域查找；找不到就向外层作用域查找，直到全局作用域。这条查找路径就是作用域链。

```js
const name = "global";

function outer() {
  const name = "outer";

  function inner() {
    console.log(name); // outer
  }

  inner();
}

outer();
```

`inner` 先找自己的作用域，没有，再找 `outer`，最后才会找全局。

## 词法作用域（静态作用域）

JavaScript 使用词法作用域：变量能否访问，由“函数定义的位置”决定，而不是“函数调用的位置”决定。

```js
const value = "global";

function show() {
  console.log(value);
}

function run() {
  const value = "local";
  show(); // 仍然是 global
}

run();
```

`show` 定义在全局位置，所以它读取的是全局的 `value`。

## 作用域与闭包的关系

闭包本质上是“函数 + 定义时的词法环境”。函数执行结束后，内部函数仍然可以访问外层作用域中的变量。

```js
function createCounter() {
  let count = 0;
  return function () {
    count += 1;
    return count;
  };
}

const counter = createCounter();
console.log(counter()); // 1
console.log(counter()); // 2
```

这里返回的函数形成了闭包，持续引用 `count`。

## 最佳实践

- 优先使用 `const`，需要重新赋值时再用 `let`
- 尽量避免 `var`，减少作用域和提升相关问题
- 减少全局变量，避免命名污染
- 变量就近声明，缩小可见范围
- 利用块级作用域隔离临时变量
