# JavaScript 中的提升（Hoisting）

## 什么是提升？

提升（Hoisting） 是 JavaScript 的一种执行机制：在代码正式运行之前，JavaScript 引擎会先进行一次编译/预处理，扫描当前作用域中所有的变量声明（var）和函数声明，并将这些声明的标识符（变量名/函数名）“提升”到其所在作用域的顶部。

提升后的变量/函数在物理代码中的位置之前就可以被访问，但值可能为 undefined或不可访问。

::: info

类比 C 语言中的函数声明和变量声明，函数和变量必须先声明后使用，否则会报错。而 JavaScript 中的提升机制允许函数和变量在声明之前就被访问，但需要注意它们的值和可访问性。

:::

## 变量提升：var

var 声明的变量会被提升，但它们的值会被初始化为 `undefined`，直到执行到赋值语句时才会被赋予实际的值。

```js{2,6}
// 输出 undefined，因为变量 test 已经被提升了，但还没有赋值
console.log(`test now is`, test);

var test = 1;
// 输出 1，test 在这里被赋值了
console.log(`test now is`, test);
```

::: warning

在现代 JavaScript 开发中，应尽量避免使用 `var`。`var` 是函数作用域（而非块级作用域），并且会发生提升并初始化为 `undefined`，这很容易导致变量在未预期的位置被访问，从而引入隐蔽 bug。推荐统一使用 `let` 和 `const`：默认使用 `const`，仅在确实需要重新赋值时使用 `let`。

:::

## 变量提升：const、let

let 和 const 声明的变量也会被提升，但它们不会被初始化，处于所谓的 **暂时性死区**（TDZ, Temporal Dead Zone）中，在声明之前访问会导致 `ReferenceError` 错误。

```js
// ReferenceError: Cannot access 'test' before initialization
console.log(`test 的值`, test); // [!code error] 报错

let test = 1; // 此处使用 const 也会有同样的错误
console.log(`test 的值`, test);
```

## 函数声明的提升

函数声明会被提升，并且整个函数体都会被提升，所以在函数声明之前就可以调用该函数。

```js{2}
// 函数 func 已经被提升了，所以可以在声明之前调用
func(); // 输出 "Hello, Hoisting!"

function func() {
    console.log(`Hello, Hoisting!`);
}
```

::: warning

需要注意的是，函数表达式的函数体不会被提升，因为它们是通过变量来引用的，而变量提升的规则不同于函数声明。

```js
// ReferenceError: Cannot access 'funcExprDemo' before initialization
funcExprDemo(); // [!code error] 报错

let funcExprDemo = function () {
  console.log(`Hello, Function Expression!`);
};
```

:::
