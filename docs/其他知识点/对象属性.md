# 对象属性

参考：[MDN - 语法与类型](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Guide/Grammar_and_types)、[MDN - 属性的可枚举性和所有权](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Guide/Enumerability_and_ownership_of_properties)

## 属性描述符

属性描述符是 JavaScript 引擎内部维护的一个对象，用于存储属性的元信息（metadata）。一个属性除了有值之外，还有一些控制它行为的标志（flags）。

平时我们写：

```js
const user = { name: "Tom" };
```

你看到的是 `name` 的值，但引擎还维护了这个属性的“配置项”（描述符），比如它：

- 能不能被改（`writable`）
- 能不能被枚举出来（`enumerable`）
- 能不能被删除或重新配置（`configurable`）

> 对象字面量创建的普通属性，默认三大标志通常都是 `true`。

我们可以通过 [`Object.getOwnPropertyDescriptor`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/getOwnPropertyDescriptor) 来查看一个属性的描述符：

```js
const user = { name: "Tom" };
console.log(Object.getOwnPropertyDescriptor(user, "name"));
// 常见输出：
// {
//   value: 'Tom',
//   writable: true,
//   enumerable: true,
//   configurable: true
// }
```

## 可写属性

一个属性是不是可写的，由 [`writable`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty#writable) 标志控制。不可写属性不能被修改。

```js
const obj = {};
Object.defineProperty(obj, "id", {
  value: 1,
  writable: false, // 不可写
  enumerable: true,
  configurable: true,
});

obj.id = 2;
console.log(obj.id); // 1（非严格模式下静默失败；严格模式下会抛错）
```

## 可枚举属性

一个属性是不是可枚举的，由 [`enumerable`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty#enumerable) 标志控制。可枚举属性会出现在 `for...in` 循环、`Object.keys`、`JSON.stringify` 等操作中。

```js
const obj = {};
Object.defineProperty(obj, "secret", {
  value: "123",
  enumerable: false, // 不可枚举
});

console.log(Object.keys(obj)); // []
console.log(obj.secret); // '123'，仍然可以直接访问
```

## 可配置属性

一个属性是不是可配置的，由 [`configurable`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Object/defineProperty#configurable) 标志控制。不可配置的属性不能被删除，也不能修改它的描述符（除了 `writable` 从 `true` 变 `false` 以外）。

> 一旦 `configurable` 设为 `false`，后续可操作空间会明显变小，要谨慎。

```js
const obj = {};
Object.defineProperty(obj, "id", {
  value: 1,
  writable: false,
  enumerable: true,
  configurable: true,
});

obj.id = 2;
console.log(obj.id); // 1（非严格模式下静默失败；严格模式下会抛错）
```

## 访问器属性

一个对象的属性可以是“数据属性”（data property）或“访问器属性”（accessor property）。它们的描述符结构不同：

- 数据属性
  核心字段是：`value`、`writable`。

- 访问器属性
  核心字段是：`get`、`set`。它没有 `value` 和 `writable`。

```js
const user = {
  firstName: "Tom",
  lastName: "Lee",
};

Object.defineProperty(user, "fullName", {
  get() {
    return `${this.firstName} ${this.lastName}`;
  },
  set(v) {
    [this.firstName, this.lastName] = v.split(" ");
  },
  enumerable: true,
  configurable: true,
});

console.log(user.fullName); // Tom Lee
user.fullName = "Jack Ma";
console.log(user.firstName); // Jack
```

> 同一个属性不能同时既是“数据属性”又是“访问器属性”。

## 4. defineProperty 的默认值是个高频坑

如果你用 `Object.defineProperty` 且不显式写标志，很多标志默认是 `false`。

```js
const obj = {};
Object.defineProperty(obj, "x", {
  value: 10,
});

console.log(Object.getOwnPropertyDescriptor(obj, "x"));
// {
//   value: 10,
//   writable: false,
//   enumerable: false,
//   configurable: false
// }
```

这和对象字面量 `{ x: 10 }` 的默认行为很不一样，面试常考。

## 5. 枚举相关：到底谁能被遍历到？

```js
const base = { inProto: 1 };
const obj = Object.create(base);
obj.own = 2;
Object.defineProperty(obj, "hidden", {
  value: 3,
  enumerable: false,
});

for (const k in obj) {
  console.log("for...in ->", k); // own, inProto（包含原型链可枚举属性）
}

console.log(Object.keys(obj)); // ['own']（仅自身可枚举属性）
console.log(Object.getOwnPropertyNames(obj)); // ['own', 'hidden']（自身所有字符串键）
```

建议在 `for...in` 中配合 `Object.hasOwn(obj, key)` 过滤原型链属性。

## 6. 对象状态控制：preventExtensions / seal / freeze

### 6.1 `Object.preventExtensions(obj)`

- 不能新增属性
- 现有属性是否可改，取决于各自标志

### 6.2 `Object.seal(obj)`

- 不能新增属性
- 所有现有属性 `configurable` 变为 `false`
- 若属性本身 `writable: true`，值仍可修改

### 6.3 `Object.freeze(obj)`

- 不能新增属性
- 所有现有属性 `configurable: false`
- 所有数据属性 `writable: false`

```js
const user = { name: "Tom", info: { age: 18 } };
Object.freeze(user);

user.name = "Jack";
console.log(user.name); // Tom

user.info.age = 20;
console.log(user.info.age); // 20（浅冻结）
```

> `freeze` 是浅层的，嵌套对象仍可变。要深冻结需递归处理。

```js
const obj = {};

Object.defineProperty(obj, "b", { value: 2, writable: false, enumerable: true });

console.log(Object.keys(obj)); // ?
obj.b = 3;
console.log(obj.b); // ?
delete obj.b;
console.log(obj.b); // ?
```
