# JavaScript 中的类

> 类这个概念在大部分面向对象编程语言中都是一个重要的组成部分，JavaScript 也不例外。虽然 JavaScript 在 ES6 之前没有正式的类定义，但开发者可以通过函数和原型链来模拟类的行为。ES6 引入了 `class` 关键字，使得创建类和实现面向对象编程变得更加直观和简洁。

## 什么是类（Class）

类是用于创建相同类型对象的 **模板** 或 **蓝图**

- 类中定义对象的“数据”（属性）和“行为”（方法）
- 通过 `new` 关键字可创建多个实例
- 多个实例结构相同，但实例数据彼此独立

## 如何定义和使用类

::: code-group

```js [Person.js]
class Person {
  constructor(name, age) {
    // 定义属性并赋值
    this.name = name;
    this.age = age;
    this.score = 0; // 可以在 constructor 中为属性赋默认值
  }

  greet() {
    // 定义方法
    console.log(`你好，我是 ${this.name}，我今年 ${this.age} 岁。`);
  }
}

const person1 = new Person("Alice", 30);
person1.greet();
console.log(person1 instanceof Person); // true
```

```ts [Person.ts]
class Person {
  // TS 需要先声明属性的类型
  name: string;
  age: number;
  score: number;

  constructor(name: string, age: number) {
    // 为属性赋值
    this.name = name;
    this.age = age;
    this.score = 0; // 可以在 constructor 中为属性赋默认值
  }

  greet() {
    // 定义方法
    console.log(`你好，我是 ${this.name}，我今年 ${this.age} 岁。`);
  }
}

const person1 = new Person("Alice", 30);
person1.greet();
console.log(person1 instanceof Person); // true
```

:::

## `new` 做了什么

当执行 `new Person("Alice", 30)` 时，核心过程可理解为：

1. 创建一个新对象
2. 将新对象的原型指向 `Person.prototype`
3. 执行 `constructor`，并把 `this` 绑定到新对象
4. 返回该对象（除非显式返回了另一个对象）

## 构造函数与 `this`

- `constructor` 用于初始化实例属性
- 类中的普通方法默认挂在 `prototype` 上，实例共享
- 方法被“单独取出”调用时，可能丢失 `this`

```js
class Counter {
  constructor() {
    this.count = 0;
  }

  inc() {
    this.count += 1;
  }
}

const c = new Counter();
const fn = c.inc;
// fn(); // TypeError: Cannot read properties of undefined（严格模式下）

fn.call(c);
console.log(c.count); // 1
```

## 继承：`extends` 与 `super`

- `extends`：子类继承父类
- `super(...)`：在子类构造函数中调用父类构造函数
- 在子类 `constructor` 中，必须先 `super(...)` 再访问 `this`

::: code-group

```js [Animal-Dog.js]
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} 发出了声音`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name);
    this.breed = breed;
  }

  speak() {
    console.log(`${this.name} 汪汪叫`);
  }
}

const dog = new Dog("旺财", "Shiba");
dog.speak();
```

```ts [Animal-Dog.ts]
class Animal {
  constructor(public name: string) {}

  speak() {
    console.log(`${this.name} 发出了声音`);
  }
}

class Dog extends Animal {
  constructor(
    name: string,
    public breed: string
  ) {
    super(name);
  }

  override speak() {
    console.log(`${this.name} 汪汪叫`);
  }
}

const dog = new Dog("旺财", "Shiba");
dog.speak();
```

:::

## 静态成员、访问器与私有字段

### 静态成员 `static`

静态属性/方法属于“类本身”，不属于实例。

```js
class MathTool {
  static version = "1.0.0";

  static add(a, b) {
    return a + b;
  }
}

console.log(MathTool.version);
console.log(MathTool.add(1, 2));
```

### 访问器 `get` / `set`

用于把“读写属性”包装为“受控行为”。

```js
class User {
  #age = 0;

  get age() {
    return this.#age;
  }

  set age(value) {
    if (value < 0) throw new Error("age 不能小于 0");
    this.#age = value;
  }
}
```

### 私有字段 `#field`

- 只能在类内部访问
- 类外直接访问会语法错误
- 比命名约定（如 `_name`）更严格

## 类与原型的关系

`class` 本质上没有改变 JavaScript 的原型继承模型：

- 实例方法定义在 `ClassName.prototype`
- 实例通过原型链访问共享方法
- 类只是让这种写法更清晰

## 常见坑

- 在子类 `constructor` 中先用 `this`、后 `super()`（会报错）
- 把实例方法解构出来直接调用，导致 `this` 丢失
- 把应放在原型上的方法写进构造函数，导致每个实例重复创建函数
- 误把静态方法当实例方法调用（如 `instance.create()`）

## 实践建议

- 优先保证“单一职责”：一个类只做一类事
- 能用组合就少用深继承（继承层级不宜过深）
- 对外暴露最少必要接口，内部细节尽量封装
- TS 项目中为公共方法补齐参数与返回值类型

## 小结

- `class` 是语法层面的增强，可读性高，但底层仍是原型机制
- 学习类不仅是记语法，更重要的是理解封装、复用与职责划分
- 先写清晰，再谈设计模式与高级抽象
