# 异步和事件循环

讨论事件循环的核心前提：**JavaScript 是一个单线程语言**。

正是因为 JavaScript 程序只有一个主线程，需要同时负责执行代码、处理事件和更新 UI，当一个任务正在执行时，其他任务必须等待，直到当前任务完成。

如果一个任务执行时间过长，就会导致页面卡顿。为了避免这种情况，JavaScript 引入了异步编程模型，使得一些耗时的操作（如网络请求、定时器等）可以在后台执行，而不会阻塞主线程。

**事件循环（Event Loop）**是 JavaScript 在单线程模型下实现异步能力的核心机制。

## 同步和异步

介绍事件循环之前，先区分一下同步和异步：

- **同步**：按顺序执行，前一个任务完成后才执行下一个任务。
- **异步**：发起任务后不阻塞主流程，结果稍后通知。

异步有几种形式：

- [回调函数](#回调函数)（Callback）
- [Promise](#promise-承诺-期约)
- [async / await](#async--await-语法糖)

## 回调函数

回调函数是最基本的异步模式，通过将一个函数作为参数传递，在异步操作完成后调用它。

```typescript
type GetDataCallback = (data: { name: string }) => void;

function getData(callback: GetDataCallback) {
  setTimeout(() => {
    const data = { name: "Alice" };
    callback(data);
  }, 1000);
}
```

在浏览器中，`setTimeout` 是一个典型的异步 API，它会在指定时间后执行回调函数，而不会阻塞主线程。所有的 DOM 事件监听器也是异步的，用户交互时触发回调函数。

```typescript
const button = document.querySelector("button");
button?.addEventListener("click", () => {
  console.log("按钮被点击了");
});
```

::: info
回调函数是实现异步的一种方式，但是回调不是只能用于异步操作，也可以用于同步场景。

比如：

```typescript
function doSomething(callback: () => void) {
  console.log("一些操作");
  callback();
}

doSomething(() => {
  console.log("执行回调");
});

// 输出：
// 一些操作
// 执行回调
```

:::

### 回调地狱

回调函数的缺点是容易导致代码层层嵌套：当数据存在线性的依赖关系时，常常会形成所谓的“回调地狱”，使得代码难以阅读和维护。

```typescript
const xhList = ["20210001", "20210002"];

// level 1
for (const xh of xhList) {
  // level 2
  getUser(xh, (user) => {
    // level 3
    const userId = user.id;
    getUserBooks(userId, (books) => {
      // level 4
      for (const book of books) {
        // level 5
        getBookComments(book.id, (comments) => {
          // level 6
          for (const comment of comments) {
            // level 7
            console.log(`用户：${user.name}`);
            console.log(`书籍：${book.name}`);
            console.log(`评论：${comment.message}`);
          }
        });
      }
    });
  });
}
```

## Promise（承诺/期约）

`Promise` 是一种更现代的异步模式，提供了链式调用和统一的错误处理机制。

### 获取一个 Promise 对象

Promise 构造函数接受一个执行器函数（executor），该函数立即执行，并接受两个参数：`resolve` 和 `reject`，分别用于标记操作成功或失败。

```typescript
const p = new Promise((resolve, reject) => {
  // 模拟异步操作
  setTimeout(() => {
    const success = true; // 模拟成功或失败
    if (success) {
      resolve("操作成功");
    } else {
      reject(new Error("操作失败"));
    }
  }, 1000);
});
```

Promise 还可以通过静态方法 `Promise.resolve` 和 `Promise.reject` 快速创建已解决或已拒绝的 Promise 对象：

```typescript
const resolvedPromise = Promise.resolve("已解决的值");
const rejectedPromise = Promise.reject(new Error("已拒绝的原因"));

console.log(resolvedPromise); // Promise {<fulfilled>: '已解决的值'}
console.log(rejectedPromise); // Promise {<rejected>: Error: 已拒绝的原因}
```

### Promise 对象的状态

一个 `Promise` 对象有三种状态：

- `pending`（进行中）：初始状态，既不是成功，也不是失败。
- `fulfilled`（已成功 / 已兑现）：操作成功完成，Promise 有一个值。
- `rejected`（已失败 / 已拒绝）：操作失败，Promise 有一个拒绝原因。

其中 `fulfilled` 和 `rejected` 合称为 `settled`（已落定 / 已敲定）

::: info
一般讨论 Promise 及其状态时，一般直接使用英文名称，哪怕是在中文语境中也是如此
:::

`Promise` 状态是不可逆的，这意味着一旦 `Promise` 从 `pending` 转变为 `fulfilled` 或 `rejected`，它就不能再改变状态了。这保证了 Promise 的可靠性和一致性。

```typescript
const p = new Promise((resolve, reject) => {
  resolve("成功");
  // reject 会执行，但是不会改变状态
  reject(new Error("失败"));
  console.log("落定之后");
});

p.then((res) => {
  console.log(p);
});

// 输出：
// 落定之后
// Promise {<fulfilled>: '成功'}
```

### Promise 的 then、catch 和 finally 方法及链式调用

上方的代码已经演示了，一个 `Promise` 可以通过 `then` 方法注册回调函数来处理成功或失败的结果。除了 `then`，还有 `catch` 用于处理拒绝的情况，以及 `finally` 用于无论成功还是失败都要执行的收尾操作。（对标错误处理的 `try...catch`）

于是上面的代码可以改写为：

```typescript
new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = Math.random() > 0.5; // 模拟成功或失败
    if (success) resolve("操作成功");
    else reject(new Error("操作失败"));
  }, 1000);
})
  .then((res) => {
    console.log("成功：", res);
  })
  .catch((err) => {
    console.error("失败：", err);
  })
  .finally(() => {
    console.log("操作完成");
  });
```

`Promise` 的 `then` 方法返回一个新的 `Promise`，使得我们可以进行链式调用

```typescript
getUser(xh)
  .then((user) => getUserBooks(user.id))
  .then((books) => getBookComments(books[0].id))
  .then((comments) => console.log(comments))
  .catch((err) => console.error(err));
```

链式调用中，`catch` 可以捕获链路中任何一个 `then` 抛出的异常。

```typescript
getUser(xh)
  .then((user) => getUserBooks(user.id))
  .then((books) => {
    // throw 是一个语句，不是表达式，所以不能直接写在箭头函数的隐式返回里
    throw new Error("获取书籍失败");
  })
  .then((comments) => console.log(comments))
  .catch((err) => console.error(err));
```

链式调用的优点一定程度上缓解了回调地狱的问题，但当依赖关系复杂，或者编写代码时未合理排布代码，仍然可能出现嵌套。

::: code-group

```typescript [未合理组织的代码]

  .then((user) => {
    console.log(user);
    return getUserBooks(user.id).then((books) => {
      if (!books[0]) return;
      console.log(books[0]);
      return getBookComments(books[0].id).then((comments) => {
        console.log(comments);
      });
    });
  })
  .catch((err) => {
    console.error(err);
  });
```

```typescript [将内部 then 转移到外部，降低嵌套层级]

    console.log(user);
    return getUserBooks(user.id);
  })
  .then((books) => {
    if (!books[0]) return;
    console.log(books[0]);
    return getBookComments(books[0].id);
  })
  .then((comments) => {
    console.log(comments);
  })
  .catch((err) => {
    console.error(err);
  });
```

### Promise 批量处理

当需要同时处理多个 Promise 时，可以使用 `Promise.all`、`Promise.allSettled`、`Promise.race` 和 `Promise.any` 来进行批量处理。

#### [`Promise.all`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise/all)

- 接收一个 Promise 可迭代对象（如数组），返回一个新的 Promise。
- 当所有 Promise 都成功时，返回一个包含所有结果的数组。

```typescript
getUser(xh)
  .then((user) => getUserBooks(user.id))
  .then((books) => {
    const commentPromises = books.map((book) => getBookComments(book.id));
    return Promise.all(commentPromises);
  })
  .then((commentsArray) => {
    commentsArray.forEach((comments, index) => {
      console.log(`书籍 ${index + 1} 的评论：`, comments);
    });
  })
  .catch((err) => {
    console.error(err);
  });
```

#### [`Promise.allSettled`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise/allSettled)

- 接收一个 Promise 可迭代对象，返回一个新的 Promise。
- 当所有 Promise 都完成（无论成功还是失败）时，返回一个包含每个 Promise 结果的数组，每个结果是一个对象，表示该 Promise 的状态和对应的值或原因。

#### [`Promise.race`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise/race)

- 接收一个 Promise 可迭代对象，返回一个新的 Promise。
- 当第一个 Promise 成功或失败时，返回那个 Promise 的结果。

#### [`Promise.any`](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Promise/any)

- 接收一个 Promise 可迭代对象，返回一个新的 Promise。
- 当第一个 Promise 成功时，返回那个 Promise 的结果；如果所有 Promise 都失败了，则返回一个 AggregateError 错误。

::: info
因为大部分情况都可以使用 `Promise.all` 所以此处只给了 `Promise.all` 的示例，其他方法的使用方式与 `Promise.all` 类似，可以参考 MDN 文档中的示例。
:::

## async / await 语法糖

`async` / `await` 是基于 `Promise` 的语法糖，使得异步代码看起来像同步代码，更易读。

### 为什么要使用 async / await

在介绍 `async` / `await` 之前，我们先考虑一个场景，还是利用之前的获取用户、书籍和评论的例子：

我们需要先获取用户信息，然后根据用户 ID 获取书籍列表，最后获取每本书的评论。我们需要依次按顺序打印出用户信息、书籍信息和评论信息，每位用户后紧跟着他们的书籍信息，每本书后紧跟着它的评论信息。

```typescript {10-11} [一种实现]
getUser(xh)

      books.map((book) => {
```

通过上面的代码，可以发现，`Promise.then` 有一个相当棘手的问题，那就是它的回调函数只能访问上一个 `then` 的结果。

如果 `Promise` 需要访问更上层的结果，就必须通过包装对象或外部变量的方式将它们传递下去。包装对象会导致这就导致了代码的层层嵌套和可读性的下降；而外部变量的写法容易产生副作用导致其他问题，不符合函数式风格。

这个问题，正是 `async` / `await` 诞生的初衷：让我们能够像写同步代码一样写异步代码，避免回调地狱和层层嵌套。

### async 函数和 await 表达式

`async` 函数是一个特殊的函数，它会隐式地返回一个 `Promise` 对象。无论函数内部返回什么值，都会被自动包装成一个已解决的 `Promise`；如果函数内部抛出错误，则会返回一个已拒绝的 `Promise`。

```typescript
async function example() {
  return "Hello, async!";
}
```

以上代码等价于：

```typescript
function example() {
  return Promise.resolve("Hello, async!");
}
```

`await` 表达式只能在 `async` 函数内部使用，它用于等待一个 `Promise` 对象的解决，并返回其结果。使用 `await` 可以让我们以同步的方式编写异步代码。

```typescript
async function example() {
  const result = await Promise.resolve("Hello, async!");
  console.log("结果：", result);
}
```

以上代码等价于：

```typescript
function example() {
  return Promise.resolve("Hello, async!").then((result) => {
    console.log("结果：", result);
  });
}
```

利用 `async` / `await`，我们可以更清晰地表达异步流程，依然以前面的获取用户、书籍和评论的例子为例：

```typescript
async function fetchData(xh: string) {
  try {
    const user = await getUser(xh);
    console.log(`用户 ${user.name}`);
    const books = await getUserBooks(user.id);
    for (const book of books) {
      console.log(`书籍 ${book.name}`);
      const comments = await getBookComments(book.id);
      for (const comment of comments) {
        console.log(`评论 ${comment.message}`);
      }
    }
  } catch (err) {
    console.error(err);
  }
}

fetchData("20210001");
```

可以看到，使用 `async` / `await` 后，代码的结构更清晰了，逻辑也更直观了。我们不再需要通过包装对象或外部变量来传递数据，也不需要担心回调地狱的问题了。

`async` / `await` 本质上是 `Promise` 的语法糖，它并没有引入全新的异步模型。核心规则为：

- `async` 函数总是返回一个 `Promise`。
- `await` 表达式的右侧会被包裹成 `Promise`（如果不是的话），然后挂起当前函数，等该 `Promise` 解决后恢复执行，并将结果作为 `await` 表达式的值返回。
- 如果 `Promise` 被拒绝，则会在 `await` 处抛出异常，可用 `try...catch` 捕获。

以下面的 `async` 函数为例：

```typescript
async function fetchData(xh: string) {
  try {
    const user = await getUser(xh);
    const books = await getUserBooks(user.id);
    const comments = await getBookComments(books[0].id);
    return { user, books, comments };
  } catch (err) {
    console.error(err);
    throw err;
  }
}
```

可以被转换为等价的 `Promise` 链式调用：

```typescript
function fetchData(xh: string) {
  return getUser(xh)
    .then((user) => {
      return getUserBooks(user.id).then((books) => ({ user, books }));
    })
    .then(({ user, books }) => {
      return getBookComments(books[0].id).then((comments) => ({ user, books, comments }));
    })
    .catch((err) => {
      console.error(err);
      throw err;
    });
}
```

可以看到，`await` 可以替换为 `.then()` 链，函数体内的多个 `await` 会形成嵌套或连续的 `then` 调用，并且原本的 `try...catch` 对应了最终的 `.catch()`。

::: warning

需要注意的是 `async` 存在“传染性”，也就是说一个 `async` 函数内部调用了另一个 `async` 函数，那么前者也必须声明为 `async`。

所以，为了支持 `async` 函数的调用链，程序顶层函数通常必须声明为 `async`，或者至少在顶层使用 `Promise` 来处理异步结果。

```typescript
async function foo() {
  const result = await bar(); // 这里调用了另一个 async 函数 bar
  console.log(result);
}

async function bar() {
  return "Hello from bar!";
}
```

:::

## 事件循环

讲了这么多异步编程的内容，最后我们来看看它们背后的核心机制：**事件循环**

事件循环是 JavaScript 在单线程模型下实现异步能力的核心机制。它负责协调执行栈、消息队列和其他异步任务之间的关系，确保 JavaScript 程序能够正确地处理异步操作。

事件循环的关键组成包括：

- **调用栈（Call Stack）**：正在执行的函数。
- **任务队列（Task Queue）**：存放待执行的任务（Task，社区俗称“宏任务”）
- **微任务队列（Micro Task Queue）**：存放优先级更高的微任务，如 `Promise.then`、`MutationObserver`、`queueMicrotask`。

在早期的 JavaScript 规范中，任务队列被分为“宏任务队列”和“微任务队列”，但现代规范已经不再使用“宏任务”这个术语，而是直接称为“任务队列”。不过为了教学方便，我们仍然可以用“宏任务”来指代那些非微任务的任务。

为了方便区分，在下文中我们仍然使用“宏任务”和“微任务”来描述不同类型的任务。

事件循环的核心执行流程如下：

1. 执行当前调用栈中的**同步代码**，直到调用栈为空。
2. 清空**整个微任务队列**（执行过程中新增的微任务也会在当前轮次清空）。
3. 从任务队列（宏任务队列）中**取出一个宏任务**并执行。
4. 重复步骤 2 和 3。

::: info

浏览器会在执行完一个宏任务并清空微任务后，视情况决定是否进行渲染（例如执行 `requestAnimationFrame` 回调、布局、绘制等）。

渲染不属于微任务，也不属于宏任务，而是在事件循环的特定时机插入。这一点对于理解 `requestAnimationFrame` 和 `requestIdleCallback` 的执行顺序很有帮助。

该简化模型可以准确解释绝大多数 JavaScript 异步代码的执行顺序。

:::

## 常见陷阱

- 忘记 `await` 导致拿到 Promise 而非结果
- `forEach` + `async` 误以为会按顺序等待
- Promise 链中遗漏 `return` 导致链路断裂
- 只写成功分支，不处理拒绝分支

### 练习 1：Promise 改造

将以下回调风格的函数 `loadUserProfile` 改造为返回 Promise 的版本。函数内部仍需要使用 setTimeout 模拟异步操作，并保留原有的延迟和错误处理逻辑（若 userId 为空或无效，回调会传入错误信息）。

```typescript
type Profile = { id: number; nickname: string; avatar: string };

function loadUserProfile(userId: number, callback: (err: Error | null, profile?: Profile) => void) {
  setTimeout(() => {
    if (userId <= 0) {
      callback(new Error("无效的用户ID"));
      return;
    }
    const profile = { id: userId, nickname: `User_${userId}`, avatar: "default.png" };
    callback(null, profile);
  }, 500);
}
```

要求：

创建一个名为 loadUserProfileAsync 的函数，它接收 userId 并返回 `Promise<Profile>`。

使用 new Promise 封装原有的回调逻辑，并在回调中调用 resolve / reject。

### 练习 2：async/await 改写 then 链

下面有一段使用 fetch（浏览器环境）获取天气信息的 then 链，请使用 async/await 将其改写，保持相同的功能和错误处理。

```typescript
function fetchWeather(cityCode: string): Promise<number> {
  return fetch(`https://api.weather.com/v1/${cityCode}`)
    .then((response) => {
      if (!response.ok) {
        throw new Error(`HTTP ${response.status}`);
      }
      return response.json();
    })
    .then((data) => data.temperature)
    .catch((err) => {
      console.error("获取天气失败:", err);
      return 20; // 默认温度
    });
}
```

要求：

改写为 async 函数，函数名为 fetchWeatherAsync。

保留原有的 try-catch 错误处理（捕获网络错误或响应非 ok 状态），并返回默认温度 20。

使用 await 等待 fetch 和 response.json()。
